exemONE  —  Improvement of One Design System

exemONE is a database performance management service with a somewhat complex concept and structure, requiring efficient design system management due to its numerous design elements. This project aimed to improve the inefficiencies and visual issues of the design system, establishing a comfortable production environment and enhancing the product's completeness.

Issues with the Existing Design System

The various design elements of exemONE are managed based on the internally built One Design System. However, the existing design system had maintenance and usability issues due to an inefficient component structure and inconsistent color palette.
Service screen of exemONE

Low Usability and Difficult Maintenance of Existing Component Structure

The existing components were not separated, requiring organization of all possible states. This structure caused inconvenience in using components and was a major factor hindering maintenance and work speed.
Existing Dropdown component with all cases organized

Modularization of Component Elements

Separated parts that could be divided into different components were modularized, and each design element used within the component was optionized. The modular design elements could be adjusted at the top component level using the Nested Instances feature, enhancing usability.
Modular component structure

Convenient Usability and Easy Maintenance

The modular structure allowed all situations to be expressed with a few clicks, making components convenient to use during design work. Additionally, with fewer variants and the need to only modify necessary parts, maintenance became much more efficient than the existing structure.
Appearance when using components with fewer variants

Structural Improvement According to Component Characteristics

Used the Instance Swap feature to structure components differently according to their characteristics. This structure significantly improved usability during design work by allowing icon changes while viewing images, rather than text, when changing the icon type of the Icon Button component.
Improved component allowing icon changes while viewing images

Improved Memory Management

Memory management was crucial for the design files of exemONE, which has many separate screens and design elements. With the improvement of all component structures, the overall number of variants decreased, greatly aiding in the memory management of the entire design file.
Improved memory management with reduced number of variants

Inefficient Color Work for Each Theme

In the existing system, when designing a screen, color work had to be done twice, once for the light theme and once for the dark theme. This doubled the work time and caused difficulties in managing both themes during design revisions due to unnecessary repetition.
Existing work method requiring color work twice for each theme

Work Efficiency Through Token Management for Each Theme

Defined all colors used in dark and light themes using the Variables feature and managed the colors of each theme as a single token. This allowed handling of color work for both themes simultaneously, saving work time and improving maintenance efficiency.
Token management using the Variables feature

Low Usability and Consistency of Existing Color Palette

The existing color palette had unnecessary stages and inconsistent brightness of colors. During design work, the process of applying several stages to find colors of the same brightness was repeated, which hindered the visual consistency of the service.
Existing color palette with inconsistent brightness at the same stage

Color Palette Allowing Clear Status Representation

The newly adjusted color palette with reduced unnecessary stages had consistent brightness at each stage, improving usability and visual consistency during design. The overall tone of the colors was also improved, allowing clear representation of system statuses.
Improved color palette with consistent tone and brightness

System Development as a Library

The improved color palette was also organized and managed as theme-specific tokens. These tokens were shared with the development team in JSON format, allowing them to be self-librarized, reducing work time and maintenance costs during development.
Theme-specific color tokens and JSON code

Design System from a Designer's Perspective

Through this project, the design system of exemONE was reorganized into a more efficient and consistent structure. Moving beyond building the system from a product perspective, identifying and enhancing system issues from the designer's perspective who directly use the system provided a new perspective on viewing design systems.
