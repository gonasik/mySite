exemONE  —  Improving the exemONE Design System

exemONE is a database performance management service with a complex structure and many design elements. Efficient management of the design system was necessary. I worked on a project to improve inefficiencies and visual issues in the design system to create a more pleasant production environment and enhance the product's quality.

Problems with the Existing Design System

exemONE’s design elements were managed using the internally developed One Design System. However, the existing system had inefficient component structures and inconsistent color palettes, leading to maintenance and usability issues during design work.
Service screens of exemONE

Low Usability and Difficult Maintenance of Existing Component Structure

The old components weren’t modular, requiring all possible states to be defined. This made them difficult to use, complicated to maintain, and slowed down the workflow.
Appearance of the existing dropdown component with all possible states defined

Modularizing Component Elements

I modularized parts that could be separated into different components and made each design element within a component optional. Using nested instances, options could be adjusted at the top-level component, improving usability.
Modular component structure

Convenient Usability and Easy Maintenance

The modular structure allowed all states to be expressed with just a few clicks, making components easier to use. With fewer variants, maintenance became more efficient compared to the old structure.
Fewer variants and appearance when using components

Improving Structure Based on Component Characteristics

Using the instance swap function, I configured structures differently based on component characteristics. For example, when changing the icon type in the icon button component, designers could see and select icons directly, greatly enhancing usability.
Improved component allowing icons to be changed by viewing images

Improved Memory Management

exemONE's design files, with many screens and elements, required good memory management. The new component structures reduced the overall number of variants, helping manage memory more effectively.
Improved memory management with reduced number of variants

Inefficient Themed Color Work

Previously, designing a screen required separate color work for light and dark themes, doubling the work time and making maintenance difficult.
Old method requiring separate work for each theme

Efficient Themed Token Management

Using variables, I defined all colors used in both dark and light themes and managed each theme's colors with a single token. This allowed simultaneous color work for both themes, saving time and improving maintenance efficiency.
Token management using the Variables feature

Low Usability and Consistency in the Old Color Palette

The old palette had too many steps and inconsistent brightness levels, making it hard to find colors with the same brightness and causing visual inconsistency in the service.
Inconsistent brightness levels of the same steps in the old color palette

Clear State Expression with a New Color Palette

The new palette had fewer steps with consistent brightness levels, improving usability and visual consistency in design. The overall tone of the colors was also improved, enabling clear expression of system states.
Improved color palette with consistent tones and brightness levels

Development Library Integration

The improved color palette was also managed as themed tokens. These tokens were shared with the development team in JSON format, allowing them to create a library, reducing development time and maintenance costs.
Color tokens for each theme and JSON code

Design System from a Designer's Perspective

Through this project, exemONE's design system became more efficient and consistent. Building the system from the perspective of the designers who use it, I identified and addressed issues, gaining new insights into viewing design systems.
